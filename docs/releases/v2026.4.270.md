## Breaking changes ‚Äî **Community Yarbo** rename

**Why:** The old domain **`yarbo`** collides with the official integration. This project now uses **`community_yarbo`** everywhere that matters.

**What you need to do:**

1. **Remove** the old community **Yarbo** config entry (Settings ‚Üí Devices & services).
2. **Delete** **`custom_components/yarbo/`** from your config (or old HACS checkout).
3. **Install** **`custom_components/community_yarbo/`** and **restart** Home Assistant.
4. **Add** **Community Yarbo** again.
5. **Update** automations, scripts, dashboards (**entity IDs**), **services** (`community_yarbo.*`), **event bus** names (e.g. **`community_yarbo_job_completed`**), and **blueprints** under **`blueprints/automation/community_yarbo/`** with **`integration: community_yarbo`** selectors.
6. **MQTT debug recordings** (if used) now live under **`community_yarbo_recordings/`**.

There is **no in-place migration** of the old config entry; remove ‚Üí install folder ‚Üí restart ‚Üí re-add is the supported path.

## Reliability and bug fixes (PR #156 and related)

### Clean shutdown and telemetry

Background work tolerates a **closed event loop** during shutdown (MQTT / thread callbacks). Retry sleeps go through a small helper instead of unguarded sleeps where appropriate. (GlitchTip **#137**; PR [#154](https://github.com/markus-lassfolk/home-assistant-yarbo/pull/154).)

### Controller timeouts

**`YarboTimeoutError`** from **`get_controller`** maps to clear **`HomeAssistantError`** / **`ServiceValidationError`** across platforms, **services**, and **repairs**, without wrongly opening **controller lost** repairs for a simple timeout. ([#147](https://github.com/markus-lassfolk/home-assistant-yarbo/issues/147))

### Missing or corrupt broker host

**`broker_host`** is **resolved** from **`broker_endpoints`** or alternate fields, **normalized** and persisted where possible, **v1 migration** can backfill, and setup raises **`ConfigEntryError`** with a clear message when the broker is unusable ‚Äî no **`KeyError`** on **`entry.data[CONF_BROKER_HOST]`**. ([#155](https://github.com/markus-lassfolk/home-assistant-yarbo/issues/155))

### Options and unload

**`_async_update_options`** and **`async_unload_entry`** validate **`hass.data[DOMAIN]`**, use safe lookups, and tolerate missing client/coordinator so stale entry IDs do not cause **`KeyError`**. ([#148](https://github.com/markus-lassfolk/home-assistant-yarbo/issues/148))

### Low battery when offline

**`battery_capacity`** may be **`None`** when the robot is offline; the low-battery path no longer compares **`None`** to integers (see **`bdc735b`** and [#146](https://github.com/markus-lassfolk/home-assistant-yarbo/issues/146)).

### Library version guard

**`MIN_LIB_VERSION`** matches the manifest floor (**`python-yarbo>=2026.3.60`**) so stale **`/config/deps/`** installs cannot pass the guard and then break on **`get_controller(timeout=...)`**. ([#153](https://github.com/markus-lassfolk/home-assistant-yarbo/issues/153))

### Failover + polling

After MQTT **failover**, polling is **restarted** when the client supports it, together with the **telemetry retry** behaviour from the coordinator work above.

### Error reporting and Sentry

- Tests stub **`yarbo.error_reporting`** so imports match production.
- The integration calls the **library‚Äôs** **`init_error_reporting(enabled=False)`** where available so **python-yarbo** does not start a duplicate Sentry client.
- **Release** tagging for GlitchTip: see **`2539a70`** above.
