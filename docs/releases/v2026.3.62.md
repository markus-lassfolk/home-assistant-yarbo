# v2026.3.62 — 2026-03-06

## Performance and "HA hangs" diagnostics

Addresses reports of Home Assistant hanging or running out of resources when the Yarbo integration is enabled, with nothing useful in logs or GlitchTip.

### Added
- **Performance diagnostics** — In **Settings → Devices & Services → Yarbo → ⋮ → Diagnostics**, the coordinator section now includes **listener_count** (number of entities) and **poll_interval**. Share these when reporting load/hang issues.
- **Performance troubleshooting** — New section in [Troubleshooting](../troubleshooting.md): **"HA hangs or runs out of resources when Yarbo is enabled"** — what to collect (diagnostics), what to try (raise telemetry interval, disable MQTT recording, raise poll interval), optional debug timing, and how to report.
- **Debug timing** — With debug logging enabled, the integration logs when `async_set_updated_data` takes more than 0.1s (with listener count) so you can see if entity updates are slow.
- **Performance test** — `tests/test_coordinator_performance.py` ensures diagnostics expose listener_count and poll_interval.

### Changed
- **Diagnostic polling** — A 0.3s delay is now applied between each of the 15 diagnostic requests (every 300s) to avoid holding the event loop and to spread load on the robot.
- **Options** — Clearer description for **Acquire controller when polling for telemetry**: when the app is closed, what "taking controller" means, and when to leave the option off.

### Fixed
- When there are more than 40 entities, a one-time INFO message suggests raising the telemetry update interval (e.g. 2–5s) if HA is slow.
